# 流向誤差之流速門檻判斷更新

更新日期：2026-05-15

## 更正重點

這次圖表討論的問題不是一般資料留存率，也不是學長 API 查詢用的 `threshold`，而是：

```text
流向誤差品質監測時，應該設定多少「最低流速門檻」才納入流向誤差判斷。
```

也就是說，流速太小時流向本身容易不穩定，若把低流速點也拿來判斷流向誤差，可能會讓流向誤差統計受到干擾。因此需要先用流速門檻篩選，再看流向誤差。

## 最新決策

```text
流向誤差的最低流速門檻採用 0.48 kn。
官方流向誤差 hotspot 採 median_dir，也就是 median_abs_dir_error。
判斷依據採 median_dir_count，而不是 mean_dir_count。
```

程式環境變數：

```text
S111_DIR_HOTSPOT_MIN_TRUTH_SPEED_KN=0.48
```

目前 `verification_pipeline.py` 的預設值已改為 `0.48`。

## 老師用語對照

本段整理老師討論中提到的統計用語，避免把「10%」、「中位數」與「0.48 kn」混在一起解讀。

| 用語 | 在本研究中的意思 | 解讀重點 |
| --- | --- | --- |
| 流速門檻 / speed threshold | 先用流速篩選資料，例如 `mean_truth_speed >= 0.48 kn` | 流速太小時，流向角度容易不穩定，因此低流速點不適合直接拿來判斷流向誤差 |
| 閾值 / threshold | 用來篩選或判斷是否超標的數值 | 這裡要分清楚：`0.48 kn` 是流速門檻；`30 deg`、`45 deg` 是流向誤差警示門檻 |
| 留存比例 / retention rate | 門檻提高後，還被保留下來的資料比例 | 老師提到接近 `10%`，是指依 `median_dir_count` 曲線選擇約保留一成資料的位置 |
| 平均值 / mean | 所有數值加總後除以資料筆數 | 資料很分散或有極端值時，平均值容易被少數大值拉高 |
| 中位數 / median | 數值排序後位在中間的值 | 比平均值更不容易受極端值影響，因此老師建議用 median 作為主要依據 |
| 標準差 / std | 表示資料分散程度 | `std` 越大，代表同一組資料內部差異越大，可用來觀察模型穩定度或離散程度 |
| 變異係數 / CV | `CV = std / mean` | 當 `CV > 1` 時，代表資料離散程度高，平均值代表性下降 |
| 極端值 / outlier | 明顯偏離大多數資料的值 | 可能把 mean 拉高，因此支持改用 median 判斷 |
| 長尾效應 / long tail | 大部分資料集中在低值，但少數資料延伸到高值 | 高流速門檻下剩下的資料會快速變少，若門檻太高，樣本可能不足 |
| `median_dir_count` | 以流向誤差中位數作為判斷基準後，符合條件的資料筆數 | 目前正式門檻選擇依據是這條留存曲線，而不是 `mean_dir_count` |

因此，老師這段話可整理為：由於資料變異係數大於 `1`，代表分布高度離散，平均值容易受極端值影響；流向誤差門檻選擇應以較穩健的 `median_dir_count` 為主要依據，並在留存比例曲線上選擇接近 `10%` 的位置，因此採用 `0.48 kn` 作為流向誤差分析的最低流速門檻。

## 0815 颱風個案是否影響門檻判斷

本次資料範圍包含 `2025-08-12 ~ 2026-04-10`，因此 `2025-08-15` 颱風個案確實有被納入全資料門檻判斷。

為確認特殊事件是否嚴重影響 `0.48 kn` 的選擇，重新以 manifest 指定的 114 個 canonical spatial-stat 檔案計算：

```text
判斷條件：
median_abs_dir_error >= 30 deg
mean_truth_speed >= 指定流速門檻
```

在 `0.48 kn` 門檻下：

```text
全部資料 direction median hotspot count      = 155,781
排除 2025-08-12 ~ 2025-08-15 後 count       = 149,702
2025-08-12 ~ 2025-08-15 貢獻 count          = 6,079
其中實際有貢獻者為 2025-08-15               = 6,079
0815 相關資料占全部比例                      = 3.90%
```

留存率比較：

```text
全部資料：
0.1 kn baseline = 1,685,787
0.48 kn count   = 155,781
retention       = 9.24%

排除 2025-08-12 ~ 2025-08-15：
0.1 kn baseline = 1,657,444
0.48 kn count   = 149,702
retention       = 9.03%
```

也就是說，0815 個案本身是較強事件，單獨看時 `0.48 kn` 留存率較高；但它在全資料中只占約 `3.90%`，排除後全體留存率只改變約 `0.21` 個百分點。因此，`0.48 kn` 的門檻選擇並未被 0815 個案嚴重主導。

可在論文中表述為：

```text
為避免特殊颱風個案主導流向誤差門檻選擇，本研究另進行排除 2025-08-12 至 2025-08-15 期間資料之敏感度檢查。結果顯示，在 0.48 kn 門檻下，全資料留存率為 9.24%，排除該期間後為 9.03%，差異僅約 0.21 個百分點，顯示此門檻對單一特殊事件並不敏感。
```

## 判斷依據

1. 閾值從 `0.2` 到 `0.4` 之間，資料留存率出現明顯斷崖式下降，代表這段是主要轉折區。
2. 老師的重點不是「中位數本身等於 0.48」，而是以 `median_dir_count` 的留存曲線作為判斷依據，抓接近 `10%` 留存比例的位置。
3. 以全資料計算時，`0.47 kn` 的留存率約為 `10.17%`，`0.48 kn` 的留存率約為 `9.24%`；若排除 `2025-08-12 ~ 2025-08-15` 颱風個案，`0.47 kn` 約為 `9.96%`，`0.48 kn` 約為 `9.03%`。
4. 因此 `0.48 kn` 可視為「接近 10% 留存、但略保守」的最低流速門檻，較能聚焦在流速較明確、流向誤差較有判斷意義的點。
5. 變異係數 `CV = std / mean` 約落在 `1.54` 到 `1.93`，大於 `1`，表示資料離散程度高，平均值容易被少數極端值拉動。
6. 因為資料呈現右偏與高離散特性，中位數比平均值更能代表主要資料群，且較不受 outlier 干擾。

## 論文可用寫法

本研究針對流向誤差品質監測另行設定最低流速門檻，以避免低流速條件下流向不穩定造成誤差統計偏移。由於不同流速門檻下的流向誤差資料留存比例呈現明顯非線性變化，尤其在 `0.2` 至 `0.4 kn` 區間出現快速下降，因此本研究以留存率曲線作為門檻選擇依據。

在比較平均值與中位數指標後，考量資料變異係數大於 `1`，顯示資料分布具高離散與偏態特性，平均值容易受到極端值影響。因此，本研究採用較具抗干擾性的 `median_dir_count` 作為主要判斷依據，並以接近 `10%` 留存比例為目標，選定 `0.48 kn` 作為流向誤差分析的最低流速門檻。此門檻可排除低流速下不穩定的流向資料，同時保留較具代表性的高流速樣本進行後續品質監測。

## 目前程式狀態

已更新：

```text
DIR_HOTSPOT_MIN_TRUTH_SPEED_KN = 0.48
```

此值會套用在方向誤差的三種 metric hotspot 統計輸出：

```text
mean_dir
median_dir
std_dir
```

其中，寫入官方 `hotspot_points` 並提供給 API 的 direction hotspot 採用：

```text
median_dir / median_abs_dir_error
```

注意：目前程式用來做流速門檻篩選的欄位是 `mean_truth_speed`。這裡的「採用 median」是指門檻決策依據採 `median_dir_count` 留存曲線，而不是已經把每格流速欄位改成 `median_truth_speed`。若後續老師明確要求用「每格 24 小時流速中位數」作為篩選欄位，才需要再新增 `median_truth_speed` 的計算與輸出。

## 尚需注意

已經產生、已經寫進資料庫的舊資料不會因為程式預設值改成 `0.48 kn` 而自動重算。若論文或 API 要完全反映這個新門檻，需要重新跑對應日期的 pipeline 或重新匯入更新後的 result package。
